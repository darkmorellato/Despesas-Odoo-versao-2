# Plano de Correção: Bug do Checklist de Pagamentos

## 📋 Descrição do Problema

Quando o usuário tenta marcar um pagamento como pago no calendário:
1. O modal de confirmação aparece corretamente
2. Ao confirmar, a notificação "Marcado como Pago!" aparece
3. O checkbox marca visualmente, mas desmarca em fração de segundos
4. O estado não persiste no Firebase

## 🔍 Análise da Causa Raiz

### Localização do Problema
**Arquivo:** `src/hooks/useCalendar.ts`

### Fluxo Atual (com Bug)

```
1. Usuário clica no checkbox
   ↓
2. ExpenseCalendar chama onToggleCheck(key, true)
   ↓
3. useCalendar.toggleCheck() é executado:
   - Define isLocalUpdate.current = true
   - Atualiza estado checks com novo valor
   ↓
4. Estado checks muda → dispara useEffect de salvamento
   ↓
5. useEffect verifica isLocalUpdate.current = true
   - Define isLocalUpdate.current = false
   - Retorna sem salvar (delay de 1s)
   ↓
6. onSnapshot (Firebase) detecta mudança no servidor
   ↓
7. onSnapshot atualiza estado checks com dados do Firebase
   ↓
8. Estado checks muda novamente → dispara useEffect de salvamento
   ↓
9. useEffect verifica isLocalUpdate.current = false
   - Inicia timer para salvar no Firebase após 1s
   ↓
10. Antes do timer completar, onSnapshot dispara novamente
    ↓
11. onSnapshot sobrescreve estado local (desmarcando o checkbox)
    ↓
12. Checkbox desmarca visualmente
```

### Problema Principal

O `isLocalUpdate.current` está sendo definido como `false` muito cedo (no início do useEffect de salvamento), antes que o salvamento no Firebase seja concluído. Isso permite que o `onSnapshot` sobrescreva o estado local antes que a atualização seja persistida.

## 🛠️ Solução Proposta

### Estratégia

Implementar um mecanismo de "debounce" que previne que o `onSnapshot` sobrescreva atualizações locais recentes, usando um timestamp para rastrear quando a última atualização local ocorreu.

### Mudanças Necessárias

#### 1. Modificar `src/hooks/useCalendar.ts`

**Adicionar nova variável de estado:**
```typescript
const lastLocalUpdate = useRef<number>(0);
```

**Modificar `toggleCheck`:**
```typescript
const toggleCheck = useCallback((key: string, status: boolean) => {
  isLocalUpdate.current = true;
  lastLocalUpdate.current = Date.now(); // Marcar timestamp da atualização
  setChecks((prev: CalendarCheck) => ({ ...prev, [key]: status }));
}, []);
```

**Modificar primeiro useEffect (onSnapshot):**
```typescript
useEffect(() => {
  if (!user) {
    setIsLoading(false);
    return;
  }

  setIsLoading(true);

  const unsubscribe = onSnapshot(
    firebaseRefs.checksRef,
    (doc) => {
      // Verificar se a atualização é muito recente (menos de 2 segundos)
      const timeSinceLastUpdate = Date.now() - lastLocalUpdate.current;
      if (timeSinceLastUpdate < 2000) {
        console.log("Ignorando atualização do Firebase (muito recente)");
        return;
      }

      if (doc.exists()) {
        const data = doc.data() as { checks?: CalendarCheck };
        setChecks(data.checks || {});
      } else {
        setChecks({});
      }
      setIsLoading(false);
    },
    (err: Error) => {
      console.error("Erro ao escutar checklist:", err);
      setIsLoading(false);
    }
  );

  return () => unsubscribe();
}, [user, firebaseRefs.checksRef]);
```

**Modificar segundo useEffect (salvamento):**
```typescript
useEffect(() => {
  if (!user || isLocalUpdate.current) {
    // Não definir isLocalUpdate.current = false aqui
    return;
  }

  const timer = setTimeout(() => {
    setDoc(firebaseRefs.checksRef, { checks }, { merge: true })
      .then(() => {
        console.log("Checklist salvo com sucesso no Firebase");
      })
      .catch((err) => {
        console.error("Erro ao salvar checklist:", err);
      });
  }, 1000);
  return () => clearTimeout(timer);
}, [checks, user, firebaseRefs.checksRef]);
```

**Adicionar useEffect para limpar isLocalUpdate:**
```typescript
useEffect(() => {
  // Limpar isLocalUpdate após 2 segundos
  const timer = setTimeout(() => {
    isLocalUpdate.current = false;
  }, 2000);
  return () => clearTimeout(timer);
}, [checks]);
```

### Fluxo Corrigido

```
1. Usuário clica no checkbox
   ↓
2. ExpenseCalendar chama onToggleCheck(key, true)
   ↓
3. useCalendar.toggleCheck() é executado:
   - Define isLocalUpdate.current = true
   - Define lastLocalUpdate.current = Date.now()
   - Atualiza estado checks com novo valor
   ↓
4. Estado checks muda → dispara useEffect de salvamento
   ↓
5. useEffect verifica isLocalUpdate.current = true
   - Retorna sem salvar (delay de 1s)
   ↓
6. useEffect de limpeza é disparado
   - Aguarda 2 segundos
   - Define isLocalUpdate.current = false
   ↓
7. Durante esses 2 segundos, onSnapshot detecta mudança
   ↓
8. onSnapshot verifica timeSinceLastUpdate < 2000
   - Retorna sem atualizar estado local
   ↓
9. Após 2 segundos, isLocalUpdate.current = false
   ↓
10. Estado checks muda → dispara useEffect de salvamento
    ↓
11. useEffect verifica isLocalUpdate.current = false
    - Inicia timer para salvar no Firebase após 1s
    ↓
12. Firebase é atualizado com sucesso
    ↓
13. Checkbox permanece marcado ✓
```

## ⚠️ Considerações Importantes

### Por que 2 segundos?
- Tempo suficiente para salvar no Firebase
- Previne conflitos com onSnapshot
- Não é perceptível para o usuário
- Pode ser ajustado se necessário

### Por que não remover isLocalUpdate?
- Ainda é necessário para evitar loop infinito
- Funciona em conjunto com lastLocalUpdate
- Fornece proteção adicional

### E se o Firebase demorar mais que 2 segundos?
- O salvamento ainda ocorrerá (não é bloqueado)
- Apenas a atualização visual será protegida
- O próximo onSnapshot trará os dados corretos
